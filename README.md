## What is CodeScan AI?

**CodeScan AI** is a zero-install, browser-based code review tool that uses GPT-4o (via OpenRouter) to analyze your code across four dimensions simultaneously:

| Dimension | What it catches |
|-----------|----------------|
| **Bugs** | Division by zero, null references, logic errors, edge cases |
| **Security** | SQL injection, hardcoded secrets, unsafe file handling, weak auth |
| **Performance** | O(nВІ) loops, redundant iterations, inefficient data structures |
| **Code Quality** | Missing type hints, poor naming, refactoring opportunities |

Paste your code or upload a file, hit **Analyze**, and get a structured report with a quality score, severity-tagged issues, fix suggestions, and corrected code snippets вҖ” all in under 10 seconds.

---

## Features

- **Syntax-highlighted editor** вҖ” custom-built with live highlighting for 13 languages, no heavy Monaco dependency
- **Animated background** вҖ” canvas-rendered node network, glowing orbs, falling code fragments, and scanline sweep
- **Structured AI report** вҖ” JSON-parsed response with score (0вҖ“100), breakdown bars, and per-issue cards
- **Expandable issue cards** вҖ” click any issue to reveal the full description, fix suggestion, and corrected code
- **File upload** вҖ” drag or select `.py`, `.js`, `.ts`, `.java`, `.cpp`, `.go`, `.rs`, and more
- **13 language selector** вҖ” Python, JavaScript, TypeScript, Java, C, C++, Go, Rust, PHP, Ruby, Swift, Kotlin, C#
- **Pure SVG iconography** вҖ” no emoji, no icon library, every icon is an inline SVG
- **`.env` key management** вҖ” API key never touches the UI, read directly from Vite's environment
- **Graceful degradation** вҖ” clear error states if the key is missing or the request fails

## Getting Started

### Prerequisites

- [Node.js](https://nodejs.org) v18 or higher
- An [OpenRouter](https://openrouter.ai/keys) API key (free tier available)

### Installation

```bash
# 1. Clone the repository
git clone https://github.com/iAdityaSharma2912/codescan-ai.git
cd codescan-ai

# 2. Install dependencies
npm install

# 3. Set up environment variables
cp .env.example .env
```

### Configure API Key

Open the `.env` file and paste your OpenRouter key:

```env
VITE_OPENROUTER_API_KEY=sk-or-v1-your-real-key-here
```

> Get your free key at [openrouter.ai/keys](https://openrouter.ai/keys). The free tier includes GPT-4o access.

### Run

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) and start reviewing code.

## How It Works

```
User pastes/uploads code
         в”Ӯ
         в–ј
Language is selected (auto-detected from file extension on upload)
         в”Ӯ
         в–ј
Prompt is constructed with language context + code
         в”Ӯ
         в–ј
POST вҶ’ openrouter.ai/api/v1/chat/completions (GPT-4o)
         в”Ӯ
         в–ј
JSON response is parsed into: score, scores{}, bugs[], security[], performance[], refactoring[]
         в”Ӯ
         в–ј
Results rendered in tabbed dashboard with expandable issue cards
```

### AI Prompt Strategy

The model is given a strict instruction to return **raw JSON only** вҖ” no markdown fences, no preamble. The response schema is fully specified in the prompt, ensuring consistent structure across all languages and code patterns.

## Security Notes

- The API key is read **only** from `import.meta.env` вҖ” it is never stored in state, localStorage, or sent anywhere except the OpenRouter API
- `.env` is excluded from git via `.gitignore`
- For production deployments, route requests through a backend proxy to fully hide the key from the client bundle

### Recommended Production Architecture

```
React Frontend
      в”Ӯ
      в”Ӯ  POST /api/review
      в–ј
Node.js / Express Backend     вҶҗ API key lives here only
      в”Ӯ
      в”Ӯ  POST openrouter.ai/api/v1/...
      в–ј
OpenRouter вҶ’ GPT-4o
```
